4  Instalación de Python e Introducción al Entorno de Desarrollo

Objetivos de aprendizaje

Al finalizar este capítulo serás capaz de:

  • Instalar Python 3.11+ correctamente en tu sistema operativo.
  • Distinguir entre los entornos de desarrollo Jupyter, Google Colab y VS Code, eligiendo el apropiado según el contexto de trabajo.
  • Comprender la diferencia funcional entre un script .py, un notebook .ipynb y un documento Quarto .qmd.
  • Crear y activar un entorno virtual para aislar las dependencias del proyecto.
  • Ejecutar tu primer script Python con aplicación directa en bioinformática.

4.1 🧬 Motivación biológica

Antes de escribir una sola línea de código, vale la pena preguntarse: ¿por qué Python y no Excel, R o MATLAB para trabajar con datos biológicos? La respuesta no es trivial. Excel carece de reproducibilidad y falla ante conjuntos de datos que superan el millón de filas —cifra modesta para un experimento de RNA-seq moderno—. R es un lenguaje estadístico excelente, pero su curva de aprendizaje en programación general es pronunciada y su rendimiento en operaciones matriciales de gran escala es inferior al de Python con NumPy. MATLAB, por su parte, es propietario y su ecosistema de biología de sistemas, aunque robusto, no alcanza la amplitud de la comunidad Python.

Python ofrece algo que ninguno de los anteriores garantiza por igual: un ecosistema abierto, reproducible y transversal. La misma instalación que utilizas para parsear archivos FASTA te permite, capítulos después, formular y resolver un problema de programación lineal con CobraPy para predecir la tasa de crecimiento de Escherichia coli bajo distintas condiciones nutricionales. Esa continuidad es el argumento definitivo.

Nota🧬 Contexto Histórico

Python fue creado por Guido van Rossum y su primera versión pública apareció en 1991. Su adopción en ciencia computacional se aceleró a partir de 2008 con la liberación de NumPy 1.0 y la posterior creación de Pandas (2009) por Wes McKinney. Hoy, según el índice TIOBE y las encuestas anuales de Stack Overflow, Python es consistentemente el lenguaje de programación más utilizado en el mundo, con una penetración especialmente dominante en ciencia de datos, aprendizaje automático y bioinformática computacional (McKinney, 2022).

4.2 1.1 Instalación de Python 3.11+

4.2.1 Windows

  1. Dirígete a https://www.python.org/downloads/ y descarga el instalador de Python 3.11 o superior para Windows (64-bit).
  2. Ejecuta el instalador. Crítico: marca la casilla “Add Python to PATH” antes de hacer clic en Install Now. Omitir este paso es el error más frecuente entre usuarios nuevos.
  3. Verifica la instalación abriendo el Símbolo del sistema (cmd) o PowerShell:
python --version
# Salida esperada: Python 3.11.x
pip --version
# Salida esperada: pip 23.x from C:\...\site-packages\pip (python 3.11)

4.2.2 macOS

macOS incluye Python 2.7 de sistema, que está obsoleto y no debe usarse. Instala Python 3 mediante Homebrew:

# Instalar Homebrew si no lo tienes:
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Instalar Python 3.11:
brew install python@3.11

# Verificar:
python3 --version

4.2.3 Linux (Ubuntu/Debian)

sudo apt update
sudo apt install python3.11 python3.11-venv python3-pip -y
python3.11 --version
Advertencia⚠️ Error Común — Conflicto de versiones

En sistemas Linux, python puede apuntar a Python 2 mientras que python3 apunta a Python 3. Siempre verifica con python3 --version. Si trabajas en un proyecto que requiere Python 3 exclusivamente, considera crear un alias en tu .bashrc:

alias python=python3
alias pip=pip3

4.3 1.2 Entornos Virtuales: aislando las dependencias

Un entorno virtual es una instalación de Python independiente y aislada del sistema. Cuando instalas CobraPy en tu entorno virtual, no afectas ninguna otra instalación de Python en tu computadora. Esta práctica es esencial en proyectos científicos porque garantiza que el manual sea reproducible: si especificas exactamente qué versiones de qué librerías usar, cualquier persona puede recrear tu entorno de trabajo.

# Crear el entorno virtual (solo una vez por proyecto):
python -m venv .venv-fba

# Activar el entorno:
# Windows:
.venv-fba\Scripts\activate
# macOS / Linux:
source .venv-fba/bin/activate

# El prompt cambiará a: (.venv-fba) usuario@maquina:~$

# Instalar las dependencias del manual:
pip install jupyter numpy pandas matplotlib seaborn cobrapy biopython

# Guardar las versiones exactas para reproducibilidad:
pip freeze > requirements.txt

El archivo requirements.txt generado lucirá así (con versiones exactas):

cobrapy==0.26.3
numpy==1.26.4
pandas==2.2.1
matplotlib==3.8.3
...

Cualquier colaborador puede instalar exactamente las mismas versiones con:

pip install -r requirements.txt

4.4 1.3 Entornos de Desarrollo: Jupyter, Colab y VS Code

4.4.1 Jupyter Notebook / JupyterLab

Jupyter es el estándar de facto para computación científica interactiva. Un notebook combina en un solo documento: celdas de código ejecutable, texto enriquecido con Markdown, ecuaciones en LaTeX y salidas gráficas. Es el entorno preferido para exploración de datos y prototipado.

# Iniciar Jupyter Lab (recomendado sobre Notebook clásico):
jupyter lab
# Se abrirá automáticamente en tu navegador en http://localhost:8888
Tip📐 Anatomía de un Notebook

Un archivo .ipynb es, internamente, un documento JSON que almacena el código fuente de cada celda, los metadatos del kernel (intérprete Python) y las salidas —incluyendo gráficas como texto base64—. Esto permite compartir resultados reproducibles sin que el receptor necesite ejecutar el código. Sin embargo, los archivos .ipynb no son amigables con Git porque los cambios en metadatos generan diffs ilegibles. Para este manual, se usa el formato .qmd de Quarto, que resuelve este problema.

4.4.2 Google Colab

Google Colaboratory es un entorno Jupyter alojado en la nube que no requiere instalación local. Proporciona acceso gratuito a GPU/TPU, lo que lo hace valioso para tareas de aprendizaje automático. Sus limitaciones son la pérdida de archivos al cerrar la sesión y la dependencia de conexión a internet.

Para usar los ejemplos de este manual en Colab, basta con copiar las celdas de código. Cuando sea necesario instalar paquetes no disponibles por defecto:

# En Colab, las instalaciones son por sesión (se pierden al reiniciar):
!pip install cobrapy --quiet

4.4.3 Visual Studio Code

VS Code es el entorno recomendado para el trabajo serio con este manual. Combina la ejecución interactiva de celdas (con la extensión Jupyter) con las ventajas de un editor de código completo: autocompletado inteligente, depuración, control de versiones integrado con Git y soporte nativo para Quarto. Los archivos .qmd se ejecutan y previsualizan directamente en VS Code con la extensión oficial de Quarto.

4.7 1.6 🔬 Ejercicio Integrador: Primer Script Biotecnológico

Ejercicio 1.1 — Nivel: Básico

El siguiente script integra los conceptos de este capítulo en un contexto biológico real. Analiza una secuencia de ADN y calcula propiedades básicas. Ejecútalo desde VS Code, Jupyter o la terminal.

# ─────────────────────────────────────────────────────────────────
# Script: analisis_secuencia.py
# Descripción: Análisis básico de una secuencia de ADN codificante
# Dependencias: ninguna (solo librería estándar de Python)
# ─────────────────────────────────────────────────────────────────

def analizar_secuencia(seq: str, nombre: str = "Secuencia") -> dict:
    """
    Calcula propiedades básicas de una secuencia de ADN.
    
    Parámetros
    ----------
    seq  : str  — Secuencia de ADN en mayúsculas (A, T, G, C)
    nombre : str — Nombre descriptivo de la secuencia
    
    Retorna
    -------
    dict con longitud, contenido GC, composición y codones de inicio
    """
    seq = seq.upper().replace(" ", "").replace("\n", "")
    
    longitud = len(seq)
    conteo   = {base: seq.count(base) for base in "ATGC"}
    gc_content = (conteo["G"] + conteo["C"]) / longitud * 100
    
    # Buscar codones de inicio ATG
    posiciones_atg = [i for i in range(len(seq) - 2) if seq[i:i+3] == "ATG"]
    
    return {
        "nombre"       : nombre,
        "longitud"     : longitud,
        "composicion"  : conteo,
        "gc_content"   : round(gc_content, 2),
        "codones_inicio": posiciones_atg,
    }


def imprimir_reporte(resultado: dict) -> None:
    """Imprime un reporte formateado del análisis."""
    print(f"\n{'='*50}")
    print(f"  REPORTE: {resultado['nombre']}")
    print(f"{'='*50}")
    print(f"  Longitud        : {resultado['longitud']} bp")
    print(f"  Contenido GC    : {resultado['gc_content']} %")
    print(f"  Composición     : {resultado['composicion']}")
    print(f"  Codones ATG     : {len(resultado['codones_inicio'])} encontrado(s)")
    print(f"  Posiciones ATG  : {resultado['codones_inicio']}")
    print(f"{'='*50}\n")


# ─── Fragmento del gen lacZ de E. coli (glucosidasa β-galactosidasa) ───
seq_lacZ = """
ATGACCATGATTACGCCAAGCTTGCATGCCTGCAGGTCGACGGATCCCCGGGAATTCGAG
CTCGGTACCCGGGGATCCTCTAGAGTCGACCTGCAGGCATGCAAGCTTGGCGTAATCAT
"""

resultado = analizar_secuencia(seq_lacZ, nombre="lacZ - E. coli (fragmento)")
imprimir_reporte(resultado)

Salida esperada:

==================================================
  REPORTE: lacZ - E. coli (fragmento)
==================================================
  Longitud        : 120 bp
  Contenido GC    : 52.5 %
  Composición     : {'A': 25, 'T': 32, 'G': 33, 'C': 30}
  Codones ATG     : 2 encontrado(s)
  Posiciones ATG  : [0, 10]
==================================================

Preguntas para reflexión: 1. ¿Por qué el contenido GC de un gen es biológicamente relevante? 2. Modifica la función para que también detecte codones de paro (TAA, TAG, TGA). 3. ¿Qué cambiarías en la función para hacerla compatible con secuencias de ARN?

El siguiente script integra los conceptos de este capítulo en un contexto biológico real. Analiza una secuencia de ADN y calcula propiedades básicas. Ejecútalo desde VS Code, Jupyter o la terminal.

```python
# ─────────────────────────────────────────────────────────────────
# Script: analisis_secuencia.py
# Descripción: Análisis básico de una secuencia de ADN codificante
# Dependencias: ninguna (solo librería estándar de Python)
# ─────────────────────────────────────────────────────────────────

def analizar_secuencia(seq: str, nombre: str = "Secuencia") -> dict:
    """
    Calcula propiedades básicas de una secuencia de ADN.
    
    Parámetros
    ----------
    seq  : str  — Secuencia de ADN en mayúsculas (A, T, G, C)
    nombre : str — Nombre descriptivo de la secuencia
    
    Retorna
    -------
    dict con longitud, contenido GC, composición y codones de inicio
    """
    seq = seq.upper().replace(" ", "").replace("\n", "")
    
    longitud = len(seq)
    conteo   = {base: seq.count(base) for base in "ATGC"}
    gc_content = (conteo["G"] + conteo["C"]) / longitud * 100
    
    # Buscar codones de inicio ATG
    posiciones_atg = [i for i in range(len(seq) - 2) if seq[i:i+3] == "ATG"]
    
    return {
        "nombre"       : nombre,
        "longitud"     : longitud,
        "composicion"  : conteo,
        "gc_content"   : round(gc_content, 2),
        "codones_inicio": posiciones_atg,
    }


def imprimir_reporte(resultado: dict) -> None:
    """Imprime un reporte formateado del análisis."""
    print(f"\n{'='*50}")
    print(f"  REPORTE: {resultado['nombre']}")
    print(f"{'='*50}")
    print(f"  Longitud        : {resultado['longitud']} bp")
    print(f"  Contenido GC    : {resultado['gc_content']} %")
    print(f"  Composición     : {resultado['composicion']}")
    print(f"  Codones ATG     : {len(resultado['codones_inicio'])} encontrado(s)")
    print(f"  Posiciones ATG  : {resultado['codones_inicio']}")
    print(f"{'='*50}\n")


# ─── Fragmento del gen lacZ de E. coli (glucosidasa β-galactosidasa) ───
seq_lacZ = """
ATGACCATGATTACGCCAAGCTTGCATGCCTGCAGGTCGACGGATCCCCGGGAATTCGAG
CTCGGTACCCGGGGATCCTCTAGAGTCGACCTGCAGGCATGCAAGCTTGGCGTAATCAT
"""

resultado = analizar_secuencia(seq_lacZ, nombre="lacZ - E. coli (fragmento)")
imprimir_reporte(resultado)
```
